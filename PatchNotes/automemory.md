## 📊 토큰 사용량 및 예상 비용 표시

AI 요약이 완료되면 생성 결과 옆에 사용량과 예상 비용이 표시됩니다.

표시 항목:

* 사용 모델
* 입력 토큰
* 일반 응답 토큰
* 추론 토큰
* 캐시 입력 토큰
* 전체 토큰
* 미국 달러 예상 비용
* 원화 환산 예상 비용

표시 예시:

```text
Gemini 3.1 Pro
· 입력 5,000
· 응답 295
· 추론 3,705
· 총 9,000토큰
· 예상 $0.0580 ≈ ₩81
```

각 API가 실제로 반환한 사용량을 기준으로 계산하며, 제공되지 않은 항목은 별도로 표시됩니다.

비용은 모델별 입력·출력·추론·캐시 단가를 구분하여 계산합니다.

### 원화 환산

* USD/KRW 환율 자동 조회
* 조회한 환율을 브라우저에 일정 시간 저장
* 환율 조회 실패 시 마지막 정상 환율 사용
* 저장된 환율이 없으면 기본 환율로 임시 계산

표시되는 금액은 유료 API 표준 단가를 기준으로 계산한 예상 금액입니다. 무료 티어, 세금, 지역별 요금 및 공급자의 반올림 정책에 따라 실제 청구액과 차이가 날 수 있습니다.

## 🔒 장기기억 슬롯 구조 인식

사용자 프롬프트의 출력 형식과 문체는 그대로 따르면서, 프로그램은 다음 구조만 장기기억 슬롯으로 인식합니다.

* `[제목]` 내부는 공백 포함 최대 10자
* 하나의 `[제목]`부터 다음 `[제목]` 직전까지를 장기기억 슬롯 하나로 인식
* 슬롯 본문 전체는 줄바꿈 포함 최대 300자

다음 요소는 강제하지 않습니다.

* 제목과 본문을 반드시 두 줄로 출력
* 본문 앞에 `-` 사용
* 슬롯 사이에 빈 줄 삽입
* 날짜 표기 방식
* 불릿, 소제목, 연표 형식
* 슬롯 개수
* 사건 병합 및 분리 기준
* 최소 본문 길이
* 문체 및 언어

따라서 다음과 같은 다양한 형식을 모두 사용할 수 있습니다.

```text
[제목] 같은 줄 본문
```

```text
[제목]
일반 본문
```

```text
[제목]
# 시대 및 연도
- 날짜별 사건
- 후속 사건
```

정상적인 결과는 프로그램이 임의로 재작성하거나 형식을 변경하지 않습니다.

제목 또는 본문이 저장 한도를 초과한 경우에만 해당 슬롯의 한도를 맞추기 위한 보정이 적용됩니다.

## 🗂️ 프롬프트 슬롯 관리

1차 장기요약 프롬프트와 2차 압축 프롬프트를 각각 여러 개의 슬롯으로 관리할 수 있습니다.

### 지원 기능

* 새 프롬프트 슬롯 추가
* 슬롯 이름 변경
* 슬롯 삭제
* 슬롯 내용 수정
* 슬롯 저장
* 드롭다운을 통한 즉시 전환
* 기본 프롬프트 불러오기
* 활성 슬롯 자동 기억
* 1차 요약과 2차 압축 슬롯 독립 관리

기존에 저장되어 있던 사용자 프롬프트가 있는 경우 별도의 사용자 프롬프트 슬롯으로 보존됩니다.

2차 압축 실행 시에는 현재 선택된 2차 압축 프롬프트 슬롯이 사용됩니다.

### 편집창 UI

* 프롬프트 종류와 슬롯 선택 영역 분리
* 불필요한 공백 제거
* 슬롯 작업을 아이콘 버튼으로 정리
* 새 슬롯 `＋`
* 이름 변경 아이콘
* 기본값 복원 아이콘
* 빨간색 삭제 아이콘
* 저장 아이콘
* 아이콘에 마우스를 올리면 기능 설명 표시
* 라이트 및 다크 테마 대응

프롬프트 편집 중에는 메인 화면에서만 필요한 API, 모델, 턴 수, 요약 스타일 및 내보내기 항목이 숨겨져 편집에 집중할 수 있습니다.

## 🪟 창 이동 및 닫기 방식 개선

모든 모달 우측 상단에 `×` 닫기 버튼이 항상 표시됩니다.

닫기 버튼은 별도의 배경이나 테두리 없이 `×`만 표시되며, 키보드로 선택했을 때만 접근성 외곽선이 나타납니다.

하위 화면의 이동 버튼은 모두 우측 하단의 `돌아가기`로 통일했습니다.

적용 화면:

* 프롬프트 슬롯 편집
* 장기기억 일괄 편집
* 2차 압축

하위 화면을 열 때 메인 창을 제거하지 않고 숨겨두기 때문에, 돌아왔을 때 다음 상태가 유지됩니다.

* 작성 중인 요약 결과
* 선택한 API 제공자
* API Key
* 모델
* 추론 설정
* 턴 수
* 요약 스타일
* 프롬프트 슬롯
* 미리보기 카드 위치
